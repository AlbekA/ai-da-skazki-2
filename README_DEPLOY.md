# Развёртывание проекта "AI да сказки" — подробная инструкция для новичка

## Что внутри
- Frontend — React + Vite (папка корня)
- Serverless функции для взаимодействия с Google GenAI — папка `netlify/functions`
- Пример .env.example — куда положить API ключи в Netlify

## Шаг 1 — подготовка кода локально
1. Установи Node.js (версия 18+ рекомендована). Скачай и установи с https://nodejs.org/
2. Клонируй репозиторий на свою машину:
   ```bash
   git clone <URL_твоего_репозитория>.git
   cd <repo-folder>
   ```
3. Установи зависимости:
   ```bash
   npm install
   ```

## Шаг 2 — запуск локально (предполагая, что у тебя нет реального ключа)
- Для локальной разработки можно использовать мок-режим или установить реальный ключ в `.env`.
- Чтобы локально протестировать без реального ключа, можно запустить только фронтенд:
  ```bash
  npm start
  ```
  и открыть http://localhost:5173

## Шаг 3 — подготовка Netlify
1. Создай аккаунт на https://netlify.com и привяжи репозиторий GitHub (настройка через UI).
2. В настройках сайта на Netlify перейди в **Site settings → Build & deploy → Environment → Environment variables** и добавь:
   - `GEMINI_API_KEY` = (тут — твой реальный Google GenAI / Gemini API ключ)
3. В разделе deploy settings оставь Build command = `npm run build` и Publish directory = `dist`. Functions directory уже настроен в `netlify.toml`.

## Шаг 4 — деплой
1. Сделай push в GitHub. Netlify автоматически запустит билд и задеплоит сайт.
2. В логах Netlify можно увидеть ошибки сборки — проверь их при необходимости.

## Важные замечания
- **Никогда** не коммить реальные секреты (ключи) в репозиторий. Храни их в Netlify (или других секретных хранилищах).
- Если у тебя нет доступа к Google GenAI, можно заменить функции на заглушки, возвращающие тестовый текст/audio.
